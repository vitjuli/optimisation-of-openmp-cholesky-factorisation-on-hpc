# Plots & Tables {#plots}

All figures are generated by `scripts/plot_bench.py` from the CSV files in `results/`.
See @ref pipeline for the full generation workflow.

```bash
python3 scripts/plot_bench.py --results results/ --outdir report/
```

## GFLOP/s vs threads

One line per matrix size n. X-axis: thread count (log₂). Y-axis: \f$n^3/(3\,t\cdot10^9)\f$.

<img src="gflops_icelake.png" alt="GFLOP/s vs threads — icelake" width="420"/>
<img src="gflops_cclake.png"  alt="GFLOP/s vs threads — cclake"  width="420"/>

Larger n saturates later and reaches higher peak because the working set fits
better in L3 cache relative to the per-step serial overhead.

## Amdahl curve fit

Measured speedup overlaid with the fitted model
\f$S_\text{Amdahl}(T)=1/\bigl(s+(1-s)/T\bigr)\f$.
Serial fraction \f$s\f$ is estimated by least-squares from the measured \f$S(T)\f$ values.

<img src="amdahl_icelake.png" alt="Amdahl fit — icelake" width="420"/>
<img src="amdahl_cclake.png"  alt="Amdahl fit — cclake"  width="420"/>

At n=4000 on icelake: \f$s\approx1.2\%\f$ (V2) and \f$s\approx0.7\%\f$ (V3).

## NUMA experiment (cclake)

`OMP_PROC_BIND=close` fills socket 0 before spilling to socket 1 (boundary at T=28);
`OMP_PROC_BIND=spread` distributes threads evenly across both sockets from the start.

<img src="numa_cclake.png"       alt="NUMA close vs spread — cclake"   width="420"/>
<img src="numa_ratio_cclake.png" alt="NUMA close/spread ratio — cclake" width="420"/>

Ratio \f$G_\text{close}/G_\text{spread}<1\f$ at T>28 confirms the NUMA hypothesis.
